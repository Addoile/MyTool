_str get_event(_str options="")

Parameters:

options - defaults to '' and may be one or more of the following:

'R'
Specifies no screen refresh.
'K'
Return keys from physical keyboard and not keyboard macro that is being played back.
'N'
Get next key. When a Slick-C® macro uses get_event for multiple key sequence key bindings, all get_event calls after the first should pass this parameter. Otherwise key stroke recording will not work since terminating key stroke recording requires that the last key sequence be removed.
'D'
Return and possibly set the prefix key delay in 10ths of seconds. To set the prefix key delay, concatenate a number to the 'D'. For example, the parameter 'D4' sets the prefix key delay to .4 seconds. The current or new prefix key delay is returned.
'B'
Reset mouse button click count to zero.
'U'
Always return unshifted key with SHIFT modifier (Ex: '<' = Shift+,).
'F'
Ignore FocusIn events coming to the command line

If 'R' option is not given, the screen is refreshed.

Returns: _str

Waits for a key to be pressed and returns a binary string representation of the key typed. For ASCII keys a string of length 1 is returned except for extended keys such as Ctrl+ENTER, Ctrl+BACKSPACE, Alt+A, Ctrl+A, etc. For extended keys a string of length 2 or 3 is returned.

Example:

     key=get_event();
     message('The key pressed was 'event2name(key));

Keyboard Functions
