int tag_tree_add_members_of(_str class_name, _str in_file_name, int tag_file_id, int &in_count)

Add the members of the given class to the tree view, as configured using tag_tree_prepare_expand. This function makes the assumption that the same qualified class name will not be used twice in one file to define separate classes. Languages having preprocessing like C++ can have this problem, the result is that the members of both classes will be merged. The number of items inserted without prompting is defined by the following interpreter variables:

def_cbrowser_low_refresh
During a refresh operation, stop at this many items.
def_cbrowser_high_refresh
First prompt at this level, if not refresh operation.
def_cbrowser_flood_refresh
Prompt at every multiple of this level.

Parameters:

class_name - Name of class to add members of, case-sensitive.

in_file_name - Only add class members located in this file, use empty string to ignore this constraint.

tag_file_id - Unique numeric ID for tag file, 0 for 'current' tag file. See tag_tree_set_user_info for more information about the use of this parameter.

in_count - (Input/Output) Number of items inserted.

Returns: int

0 on success, < 0 on error.

Category:

Tagging Functions
