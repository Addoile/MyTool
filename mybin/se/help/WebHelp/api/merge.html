_command int merge(_str arg1='')

Displays 3 Way Merge Setup dialog box. A 3 Way Merge is typically used after two people make a local copy of the same source file and make some modifications to their local copy. A 3 Way Merge very cleverly takes both sets of changes and creates an new source file. If there are any conflicts (because they edited the same area of code), a dialog box is displayed which lets you select whose change you want in the output file. Our 3 Way Merge lets you edit the output file without exiting 3 Way Merge mode.

Command line usage: merge [options] <base> <rev1> <rev2> <out>

Command line options(case insensitive):

  • -bb - Base is a buffer name and should be loaded +b w/o absolute
  • -bbi - Base is a buffer id and should be loaded +b w/o absolute
  • -b1 - rev1 is a buffer name and should be loaded +b w/o absolute
  • -b1i - rev1 is a buffer id and should be loaded +b w/o absolute
  • -b2 - rev2 is a buffer name and should be loaded +b w/o absolute
  • -b2i - rev2 is a buffer id and should be loaded +b w/o absolute
  • -bout - output is a buffer name, and should not be deleted when the merge dialog is closed(saved for the user).
  • -bouti - output is a buffer id, and should not be deleted when the merge dialog is closed(saved for the user).
  • -smart - same as the "use smart merge" button on the dialog (uses the diff to try to resolve some conflicts)
  • -interleaved - same as turning on "interleaved output" on dialog
  • -q - quiet option:only shuts off "Merge complete x conflicts" message
  • -callersaves - If this option is used, the file is not actually saved after the user is prompted. A 1 is returned if the user chose to save the file. Returns 2 if the user chose not to save the file.

    Returns: int

    If successful, returns: 1 if there were conflicts. if callersaves option is specified, returns 1 or 2. 0 if no conflicts. Otherwise, a negative error code is returned. Note that "successful" implies nothing about the number of differences between the files, just that all files were loaded successfully, and the merge engine ran succesfully.
