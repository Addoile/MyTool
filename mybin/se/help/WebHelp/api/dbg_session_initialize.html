int dbg_session_initialize(int session_id, _str host_or_program, _str port_or_pid, _str arguments, int timeout, _str debugger_path, _str debugger_args, ...)

Initialize the connection to the debugger

Parameters:

session_id - debugger session ID (1 .. max_session)

host_or_program - The usage of this argument may vary between debuggers and types of connections. For example:

  • host -- Name of host to connect to. If the host name is the empty string, assume 'localhost'
  • program -- Program to debug. If we are attaching to a port this parameter is still useful becuase it is needed for loading debugging information

port_or_pid - The usage of this argument may vary between debuggers and types of connections. For example:

  • port -- Port to connect to. This corresponds to the host parameter above. Default is 8000
  • pid -- Process ID to attach to

arguments - Command line arguments to pass to program

timeout - Timeout for connection and debugger communication in milliseconds

debugger_path - Path to debugger executable

debugger_args - Additional arguments needed to invoke debugger

working_dir - Working directory to run program in

Returns: int

0 on success, <0 on error.

Since:

10.0

Example:

To connect to the local host on port 8000 with a 10 second timeout: 

   int status = dbg_session_initialize(session_id, "", 8000, 10000, NULL, NULL);
   if (status) {
      return status;
   }

To debug the 'hellow" program and pass it arguments "1 2 3 4" with a 10 second timeout: 

   int status = dbg_session_initialize(session_id, "hellow", 0, "1 2 3 4", 10000, NULL, NULL);
   if (status) {
      return status;
   }
