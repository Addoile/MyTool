int tag_tree_insert_fast(int tree_wid, int tree_index, int match_type, int local_or_context_id, int include_tab, int force_leaf, int tree_flags, int include_sig, int include_class, ...)

This optimized version of tag_tree_insert_tag is used to quickly insert an item from the current tag database, context, locals, or match set into a tree control at the specified location. Use tag_tree_prepare_expand prior to calling this function in order to configure the browser bitmap set.

Parameters:

tree_wid - Window ID of the tree control to insert into.

tree_index - Parent tree node index to insert item under.

match_type - Specifies where the tag to insert comes from:

VS_TAGMATCH_tag
current tag from database
VS_TAGMATCH_context
item in current context (buffer)
VS_TAGMATCH_local
item in local variable set (function)
VS_TAGMATCH_match
item in match set

local_or_context_id - 0 if inserting current tag from database, otherwise, specifies the unique integer ID of the item from the context, locals, or match set, as specified by match_type.

include_tab - append class name after signature if 1, prepend class name with :: if 0. See tag_tree_make_caption for further explanation.

force_leaf - If < 0, force leaf node, otherwise choose by tag type. Normally "container" tag types, such as classes or structs are automatically inserted as non-leaf nodes.

tree_flags - Flags passed to _TreeAddItem.

include_sig - include function/define/template signature

include_class - include class name

user_info - Per-node user data for tree control

Returns: int

tree index on success, <0 on error.

Category:

Tagging Functions
