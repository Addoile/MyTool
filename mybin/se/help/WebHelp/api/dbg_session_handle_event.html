int dbg_session_handle_event(int session_id, _str &event_kind, int &event_flags, int &event_thread, _str &event_class, int &breakpoint_id)

Handle the next event from this debugging session's event queue. This function returns '0' if the event was handled and disposed of. Otherwise, it will return '1', indicating that you need to handle the event yourself.

Parameters:

session_id - debugger session ID (1 .. max_session)

event_kind - (reference) kind of event, including:

  • "step"
  • "breakpoint"
  • "frame_pop"
  • "exception"
  • "user"
  • "thread_start"
  • "thread_stop"
  • "class_prepare"
  • "class_unload"
  • "class_load"
  • "field_access"
  • "field_modify"
  • "catch"
  • "method_entry"
  • "method_exit"
  • "vm_start"
  • "vm_stop"

event_flags - (reference) Event handling flags, what needs updating?

  • VSDEBUG_EVENT_SUSPEND_ALL
  • VSDEBUG_EVENT_SUSPEND_THREAD
  • VSDEBUG_EVENT_GOTOPC
  • VSDEBUG_EVENT_CHANGE_THREAD
  • VSDEBUG_EVENT_UPDATE_CLASSES
  • VSDEBUG_EVENT_UPDATE_REGS
  • VSDEBUG_EVENT_UPDATE_MEMORY
  • VSDEBUG_EVENT_UPDATE_THREADS
  • VSDEBUG_EVENT_UPDATE_STACK
  • VSDEBUG_EVENT_UPDATE_VARS

event_thread - (reference) ID of thread event occurred in

event_class - (reference) Class that event occurred in

breakpoint_id - (reference) ID of breakpoint for breakpoint events

Returns: int

0 on success, <0 on error.

Since:

10.0

Category:

Debugger Functions
