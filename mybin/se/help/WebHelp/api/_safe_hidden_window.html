void _safe_hidden_window()

Helps to ensure that buffer cursor positions are not lost. Buffer positions can be lost when you switch buffers when a hidden window is active. Before you switch buffers when the hidden window is active, call this function.

Switching buffers while the hidden window is active can cause the following problems:

  • If a view other than the VSWID_HIDDEN is active, and the original buffer is not restored, a system macro may start modifying a buffer the user is editing.

  • If the hidden window already has a buffer that the user is editing active, the old buffer position information may get destroyed.
