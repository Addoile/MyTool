We use the term method to indicate a function which takes a SlickEdit object instance handle as the first argument.

We use the term property to indicate one of the VSP_??? constants defined in "vs.h". The following functions are used to get and set properties:

vsPropSetB
Set buffer (array of bytes) property.
vsPropGetB
Get buffer (array of bytes) property.
vsPropSetI
Set integer or boolean property.
vsPropGetI
Get integer or boolean property.
vsPropSetL
Set long integer property.
vsPropGetL
Get long integer property.
vsPropSetZ
Set null terminated string property.
vsPropGetZ
Get property as null terminated string.

These property functions automatically do conversions from one type to another where possible. For example, you can use the vsPropGetZ function to get an integer property such as the line number (VSP_LINE) in string format. An error will occur if the property can not be converted to the actual type.

Note: To reduce the on-line help, we show the Slick-C names for properties. The Slick-C name for a property can be converted to one of the VSP_??? constants by changing the leading "p" to VSP, uppercasing all characters, and removing all underscores except the first underscore. For example, the Slick-C property name p_buf_name corresponds to VSP_BUFNAME.
