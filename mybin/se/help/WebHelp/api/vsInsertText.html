int VSAPI vsInsertText(int wid, const char *pBuf, int BufLen=-, int Binary=a, unsigned char NLCh1='\r', unsigned char NLCh2='\n')

Insert string at cursor position. You may use this function to insert multiple lines of text. NLCh1 and NLCh2 define a 1 or two character end of line sequence. You must specify NLCh1==NLCh2 if you want a 1 character end of line sequence. The end of line sequence is used to parse the source string pBuf into multiple lines. The NLCh1 and NLCh2 are ignored if the binary option is non-zero.

When you use this function on a record file (p_buf_width!=0) and the binary argument is non-zero, all data is inserted into the current line and not broken up into multiple lines.

NOTE: If you are inserting at the end of a file and the last character of pBuf is a new line, you can end up with a blank line that has no new line characters in it. You can test for this condition by calling vsQLineLength(0,1) which will return 0. Use vsDeleteLine(0) to delete the line.

Parameters:

wid - Window id of editor control. 0 specifies the current object.

pBuf - Text to insert. May contain line separator characters.

BufLen - Number of characters in pBuf. -1 specifies that pBuf is null terminate and length can be determined from it.

binary - Specify a non-zero value for the binary argument if you want to allow bisection of newline characters (i.e. CRLF).

NLCh1 - First character of end of line sequence.

NLCh2 - Second character of end of line sequence.

Returns: int

Returns 0 if successful.

Applies To:

Edit Window, Editor Control

Categories:

Edit Window Methods, Editor Control Methods
