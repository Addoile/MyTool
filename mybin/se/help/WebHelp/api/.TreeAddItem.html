int _TreeAddItem(int ItemIndex, _str caption, int flags=0, int NonCurrentBMIndex=0, int CurrentBMIndex=0, int ShowChildren=TREE_NODE_EXPANDED, int moreFlags=0, typeless userInfo=0)

Adds a new sibling or child item.

Parameters:

ItemIndex - Specifies a tree item. The root of the tree can't be deleted and always has an item index of 0.

caption - caption to be used for new tree node.

flags - One or more of the following flags ORed together.

TREE_ADD_BEFORE
Add this item before the item specified.
TREE_ADD_AS_CHILD
Add this item after last child of the item specified by ItemIndex.
TREE_ADD_SORTED_CS
Add this item sorted case sensitive.
TREE_ADD_SORTED_CI
Add this item sorted case insensitive.
TREE_ADD_SORTED_FILENAME
Add this item sorted by filename.

NonCurrentBMIndex - Names table index to a picture which gets displayed to the left of the caption text when this node is not the current node. We recommend you specify the same values for NonCurrentBMIndex and CurrentBMIndex for leaf items.

CurrentBMIndex - Names table index to a picture which gets displayed to the left of the caption text when this node is the current node.

ShowChildren - whether to show the children of this node. TREE_NODE_COLLAPSED specifies not to show the children. Specify TREE_NODE_EXPANDED to initially show children of this node. Set this to TREE_NODE_LEAF for leaf items.

moreFlags - Combination of TREENODE_* flags.

userInfo - you to specify per-node user data. See _TreeSetUserInfo.

Returns: int

Returns the item index of the new item.

Example:

#include "slick.sh"

defeventtab form1;
void tree1.on_create()
{
   flag=TREE_ADD_AS_CHILD;


container1=prev=_TreeAddItem(0,"Container1",TREE_ADD_AS
_CHILD, _pic_fldclos, _pic_fldaop,0);
   for (i=0;i<3;++i) {
      _TreeAddItem(container1,"item "i,TREE_ADD_AS_CHILD,
_pic_file, _pic_file,-1);
   }
   container2=_TreeAddItem(container1,"Container2",0,
_pic_fldclos, _pic_fldaop,0);
   for (i=0;i<3;++i) {
      _TreeAddItem(container2,"item "i,TREE_ADD_AS_CHILD,
_pic_file,_pic_file,-1);
   }
}
