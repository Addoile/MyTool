int VSAPI vsOpenTempView(const char *pszFilename, int *pTempWindowId, int *pOrigWindowId=a, const char *pszLoadOption=a, const char *pszOpenOption=a)

This function creates an editor window which is linked to an existing or new buffer. Use this function when you want to do editor processing on a file but you do not want a visible window created. Use the vsActiveWindow and/or the VSP_WINDOWID property to switch windows. The call vsPropGetI(0,VSP_WINDOWID) returns the current window id.

Parameters:

pszFilename - Name of file to process.

pTempWindowId - Set to newly created window id.

pOrigWindowId - Set to current window id. There is always a valid current window id.

pszLoadOption - File load options

""
Display existing buffer or read pszFilename from disk.
"+bi buf_id>"
Ignore pszFilename and create new window of buffer corresponding to buf_id.
"+d"
Always read pszFilename from disk even if this file is already loaded in the editor.
"+b"
Only succeed if a buffer already exists of the name pszFilename.

pszOpenOption - File open options

C
Clear the buffer.
M
Select the editing mode for this file.
X
If the file is not found, a new file is created.
L
Specifies that select edit mode will be called later, but the file should have the correct load options initialized.
T
Throw away changes when temp view is closed.

Returns: int

If successful, returns 0 and the temp window/buffer is active. Otherwise a negative error code is returned.

Categories:

Buffer Functions, File Functions
