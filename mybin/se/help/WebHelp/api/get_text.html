_str get_text(int count=1, int seek_pos=-3)

Parameters:

count - Specifies the number of bytes to get. The following negative numbers have special meaning:

-1
Get current unicode character or DBCS character
-2
Get composite character or DBCS character

seek_pos - Specifies the byte offset in the file to get the bytes from. -3 specifies the cursor position.

Returns: _str

Returns count characters from current buffer starting from seek_pos. get_text is a stream-oriented function which for ASCII buffers will return line separation characters. Use the get_text function to retrieve strings found by a regular expression search. count defaults to 1 if not specified. seek_pos defaults to the physical character at the cursor position or the first physical character to the left of the cursor.

Applies To:

Edit_Window, Editor_Control

Example:

        // This example works for files with tab characters.
        status=search('{if|then|while}','rew');
        if (status) {
             message('Match not found');
             return(status);
        }
        word=get_text(match_length('0'),match_length('S0'))
