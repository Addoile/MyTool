void save_search(_str &search_string, int &flags, _str &word_re, _str &ReservedMore="", int &flags2=0)

Saves the current search data we recommend using the ReservedMore variable to ensure all search data is restored. For backward compatibility, the ReservedMore parameter is optional. The built-in function restore_search is called with the same data to restore the search which is used by the built-in functions repeat_search, _select_match, and match_length.

Parameters:

Flags - is a combination of the search flag constants below (defined in "slick.sh"):

  • VSSEARCHFLAG_IGNORECASE
  • VSSEARCHFLAG_MARK
  • VSSEARCHFLAG_POSITIONONLASTCHAR
  • VSSEARCHFLAG_REVERSE
  • VSSEARCHFLAG_RE
  • VSSEARCHFLAG_WORD
  • VSSEARCHFLAG_UNIXRE
  • VSSEARCHFLAG_NO_MESSAGE
  • VSSEARCHFLAG_GO
  • VSSEARCHFLAG_INCREMENTAL
  • VSSEARCHFLAG_WRAP
  • VSSEARCHFLAG_HIDDEN_TEXT
  • VSSEARCHFLAG_SCROLL_STYLE
  • VSSEARCHFLAG_BINARYDBCS
  • VSSEARCHFLAG_BRIEFRE
  • VSSEARCHFLAG_PRESERVE_CASE
  • VSSEARCHFLAG_WORDPREFIX
  • VSSEARCHFLAG_WORDSUFFIX
  • VSSEARCHFLAG_WORDSTRICT
  • VSSEARCHFLAG_HIDDEN_TEXT_ONLY
  • VSSEARCHFLAG_NOSAVE_TEXT
  • VSSEARCHFLAG_NOSAVE_TEXT_ONLY
  • VSSEARCHFLAG_PROMPT_WRAP

flags2 - currently contains flags for color coding. Constants are not yet defined for this.

Example:

search('test');
save_search(string, options, word_re);
status=search('xyz');
if (status) {
    messageNwait("Can't find xyz");
}
restore_search(string, options, word_re);
// Repeats search for 'test'
repeat_search();
