_command int ispf_move(_str arglist="")

The MOVE command moves the contents of a file or PDS member into the data being edited. In addition to the file being copied, you can specify the following on the command line:

Syntax:

   MOVE [member] [AFTER label ] [BEFORE label ]

Arguments:

AFTER label
The destination for the data being copied. The label may be either an ISPF label or a line number. The data will be inserted after the specified line.
BEFORE label
The destination for the data being copied. The label may be either an ISPF label or a line number. The data will be inserted after the specified line.

If neither AFTER nor BEFORE is specified, the default location to insert the data is after the current line (cursor position). However, if there is an A or B line command in the prefix area, the data will be inserted at that point.

This command is very similar to the ispf_copy command, except that instead of copying the data, this command will copy the data and then remove the file the data is copied from, which is NOT undoable, thus this needs to be used with care

Returns: int

Returns 0 if successful. Common return codes are FILE_NOT_FOUND_RC, PATH_NOT_FOUND_RC, TOO_MANY_SELECTIONS_RC, and TOO_MANY_FILES_RC. On error, a message is displayed.

Category:

ISPF Primary Commands

ispf_copy, ISPF Line Command A, ISPF Line Command B, ISPF Line Labels
