int tag_check_context()

Check if the current buffer position is still within the current context

For synchronization, threads should perform a tag_lock_context(false) prior to invoking this function.

Returns: int

One of the following codes:

0
The context is not set or totally wrong (maybe changed buffers)
-1
The context info loaded, but the cursor is out of "current" context
1
The context is within the tag definition
2
The context is within the scope of the tag/function

Category:

Tagging Functions
