_str qsearch(_str options, _str incremental="")

Prompts for a string to search for. The options are the same as those supported by the find command and are displayed in a more English-like form in the message prompt. Unlike the find command, this function does not use any default options not specified by the options parameter and always prompts for the search string argument so that no delimiters are required. The options argument is a string of one or more of the following:

E
Exact case.
I
Ignore case.
-
Reverse search.
M
Limit search to marked area.
<
If found, place cursor at beginning of word.
>
If found, place cursor at end of word.
R
Interpret string as a SlickEdit regular expression. See section SlickEdit Regular Expressions for syntax of regular expression.
U
Interpret string as a UNIX regular expression. See section UNIX Regular Expressions.
B
Interpret string as a Brief regular expression. See section Brief Regular Expressions.
H
Search through hidden lines.
N
Do not interpret search string as a regular search string.
P
Wrap to beginning/end when string not found.
W
Limits search to words. Used to search for variables.

W=SlickEdit-regular-expression
Specifies the valid characters in a word. The default value is [A-Za-z0-9_$].

W:P
Limits search to word prefix. For example, searching for "pre" matches "pre" and "prefix" but not "supreme" or "supre".

W:PS
Limits search to strict word prefix. For example, searching for "pre" matches "prefix" but not "pre", "supreme" or "supre".

W:S
Limits search to word suffix. For example, searching for "fix" matches "fix" and "sufix" but not "fixit".

W:SS
Limits search to strict word suffix. For example, searching for "fix" matches "sufix" but not "fix" or "fixit".

Y
Binary search. This allows start positions in the middle of a DBCS or UTF-8 character. This option is useful when editing binary files (in SBCS/DBCS mode) which may contain characters which look like DBCS but are not. For example, if you search for the character 'a', it will not be found as the second character of a DBCS sequence unless this option is specified.

,
Delimiter to separate ambiguous options.

XCCLetters
Requires the first character of search string NOT be one of the color coding elements specified. For example, "XCS" requires that the first character not be in a comment or string. CCLetters is a string of one or more of the following color coding element letters:

O
Other
K
Keyword
N
Number
S
String
C
Comment
P
Preprocessing
L
Line number
1
Symbol 1
2
Symbol 2
3
Symbol 3
4
Symbol 4
F
Function color
V
No save line

CCCLetters
Requires the first character of search string to be one of the color coding elements specified. See CCLetters above.

While you enter the string parameter the following keys take on a new meaning:

Ctrl+R
Searches in reverse for the next occurrence of the search string
Ctrl+S
Searches forward for the next occurrence of the search string.
Ctrl+T
Toggles regular expression pattern matching on/off. See section Regular Expressions for information on regular expressions. The key bound to the BRIEF emulation command re_toggle will also toggle regular expression pattern matching.
Ctrl+W
Toggles word searching on/off.
Ctrl+C
Toggles case sensitivity. The key bound to the BRIEF emulation command case_toggle will also toggle the case sensitivity.
Ctrl+O
Toggles incremental search mode.
Ctrl+Q
Quotes the next character typed.

A non-null value for incremental argument may be specified if you want searching to take place as the search string is entered.

Returns: _str

Returns 0 if the search string specified is found. Common return codes are STRING_NOT_FOUND_RC, INVALID_OPTION_RC, and INVALID_REGULAR_EXPRESSION_RC. On error, message is displayed.

Applies To:

Edit Window
