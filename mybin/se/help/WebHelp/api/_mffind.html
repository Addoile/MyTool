int _mffind(_str search_string, _str options, _str files, _str notused_atbuflist, int mfflags=0, boolean searchProjectFiles=false, boolean searchWorkspaceFiles=false, _str wildcards='', _str file_exclude='', boolean files_delimited_with_pathsep=false, int grep_id=0)

Searches for occurrences of the string, search_string, in the files and buffers specified. When an occurrence is chosen, the file is opened and the cursor is placed on the occurrence. Use the find_next (Ctrl+G) and find_prev commands to find the next and previous occurrences respectively.

Parameters:

search_string - String to search for.

options - A string of zero or more of the following search options:

E
Exact case.
I
Ignore case.
-
Reverse search.
M
Limit search to marked area.
<
If found, place cursor at beginning of word.
>
If found, place cursor at end of word.
R
Interpret string as a SlickEdit regular expression. See section SlickEdit Regular Expressions for syntax of regular expression.
U
Interpret string as a UNIX regular expression. See section UNIX Regular Expressions.
B
Interpret string as a Brief regular expression. See section Brief Regular Expressions.
H
Search through hidden lines.
N
Do not interpret search string as a regular-expression search string.
P
Ignored.
W
Limits search to words. Used to search for variables.
W=SlickEdit-regular-expression
Specifies the valid characters in a word. The default value is [A-Za-* z0-9_$].
W:P
Limits search to word prefix. For example, searching for "pre" matches "pre" and "prefix" but not "supreme" or "supre".
W:PS
Limits search to strict word prefix. For example, searching for "pre" matches "prefix" but not "pre", "supreme" or "supre".
W:S
Limits search to word suffix. For example, searching for "fix" matches "fix" and "sufix" but not "fixit".
W:SS
Limits search to strict word suffix. For example, searching for "fix" matches "sufix" but not "fix" or "fixit".
Y
Binary search. This allows start positions in the middle of a DBCS or UTF-8 character. This option is useful when editing binary files (in SBCS/DBCS mode) which may contain characters which look like DBCS but are not. For example, if you search for the character 'a', it will not be found as the second character of a DBCS sequence unless this option is specified.
,
Delimiter to separate ambiguous options.
XCCLetters
Requires the first character of search string NOT be one of the color coding elements specified. For example, "XCS" requires that the first character not be in a comment or string. CCLetters is a string of one or more of the following color coding element letters:
O
Other
K
Keyword
N
Number
S
String
C
Comment
P
Preprocessing
L
Line number
1
Symbol 1
2
Symbol 2
3
Symbol 3
4
Symbol 4
F
Function color
V
No save line
CCCLetters
Requires the first character of search string to be one of the color coding elements specified. See CCLetters above.

files - Zero or more space delimited file specifications which may contain wild cards. '+T' and '-T' switches may be embedded between files specifications to turn subdirectory (tree) searching on or off.

atbuflist - Zero or more space delimited buffer names. If a buffer name starts with a '@' character, it is assumed to be a buffer which contains a list of buffer names to search.

mfflags - One or more of the following flags:

MFFIND_CURBUFFERONLY
Search current buffer only.
MFFIND_FILESONLY
List files only.
MFFIND_APPEND
Append to current output.
MFFIND_MDICHILD
Output to MDI child.
MFFIND_SINGLE
Stop after first occurrence.
MFFIND_GLOBAL
Find all without prompting.

Returns: int

Returns 0 if one or more occurrences are found.

Example:

// Case sensitive word search for "main"  through all c and h files in or
below the current
// directory.
_mffind('main', 'ew', '+t *.c *.h','');

Applies To:

Edit Window

Category:

Search Functions
