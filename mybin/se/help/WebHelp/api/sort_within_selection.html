_command typeless sort_within_selection(_str cmdline='')

Sorts the text within the selection in ascending order. If a character selection is used, it is converted into a line type selection.

Multiple options having the following meaning may be specified:

A
Sort in ascending order.
D
Sort in descending order.
I
Case insensitive sort (Ignore case).
E
Case sensitive sort (Exact case).
U
Remove duplicate lines.
-N
Sort numbers
-F
Sort filenames
-FN
Sort filenames by name part only
-FC
Sort filenames as strings using file case sensitivity

Command line examples:

sort-within-selection
Sort in ascending order and exact case.
sort-within-selection I
Sort in ascending order and ignore case.
sort-within-selection DI
Sort in descending order and ignore case.
sort-within-selection -n d
Sort numbers in descending order

Parameters:

cmdline - is a string in the format: [ A | D] [ E | I ] [U] [ -N | -F | -FN | -FC ]

Returns: typeless

Returns 0 if successful. Common return codes are INVALID_OPTION_RC, LINE_OR_BLOCK_SELECTION_REQUIRED_RC, NOT_ENOUGH_MEMORY_RC, TOO_MANY_SELECTIONS_RC, and INVALID_SELECTION_HANDLE_RC. On error, message displayed.

Applies To:

Edit Window, Editor Control

Category:

Selection Functions

See Also:

sort_buffer, sort_on_selection, gui_sort
