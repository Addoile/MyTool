int VSAPI vsMacroQRecordSource()

Use vsMacroQRecordSource and the vsMacroQRecording function to determine whether your macro command needs to generate source code. If you have written a command, you should use vsMacroQRecordSource to check whether you need to generate source. If your command executes another command with vsExecute or vsCallIndex you may want to use vsMacroSetRecordSource to turn on or off macro recording before making the call.

Returns: int

Returns non-zero value if a macro is currently being recorded. This will return 0 if the pause_recording command is called to temporarily turn off recording.
