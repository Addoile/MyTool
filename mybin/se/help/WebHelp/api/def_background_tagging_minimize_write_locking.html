boolean def_background_tagging_minimize_write_locking

Specifies that the background tagging engine should using an alternate technique to write to the database in order to minimize the amount of time the database is locked for writing. This technique requires more memory because it caches copies of database blocks in memory while updating the database. This allows the main thread to continue reading the un-modified version of the database and the changes to be quickly swapped in after all the records have been updated in the background.

While there is no set limit on how much more memory this option uses, in general, it will require an additional 10-50M while background tagging is running.

Default:

true

Category:

Configuration Variables
