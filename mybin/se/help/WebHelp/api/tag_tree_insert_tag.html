int tag_tree_insert_tag(int tree_wid, int tree_index, int include_tab, int force_leaf, int tree_flags, _str tag_name, _str type_name, _str file_name, int line_no, _str class_name, int tag_flags, _str signature, ...)

API function for inserting a tag entry with supporting info into the given tree control.

Parameters:

tree_wid - Window ID of the tree control to insert into.

tree_index - Parent tree node index to insert item under.

include_tab - append class name after signature if 1, prepend class name with :: if 0. See tag_tree_make_caption for further explanation.

force_leaf - if < 0, force leaf node, otherwise choose by type. Normally "container" tag types, such as classes or structs are automatically inserted as non-leaf nodes.

tree_flags - flags passed to _TreeAddItem.

tag_name - Name of tag (symbol) to add to database.

type_name - Tag type name. See tag_get_type for a list of all standard tag types. Generally the tag type is a single word, all lowercase. If the string given is not a standard type, a new user defined type will be inserted into the database.

file_name - Path to file that is located in

line_no - Line number that tag is positioned on. Use 0 to represent an unknown line number.

class_name - Name of class or package or other container that the tag belongs to. See VS_TAGSEPARATOR_class and VS_TAGSEPARATOR_package for more details about constructing this value. If the tag is not in a class scope, simple pass the empty string for this value.

tag_flags - Tag flags indicating symbol attributes, see tag_insert_simple for details.

signature - Tag signature, includes type, value, and arguments for the symbol being inserted. See VS_TAGSEPARATOR_args and VS_TAGSEPARATOR_equals for more details about constructing this value.

user_info - Per-node user data for tree control

Returns: int

tree index of new item on success, <0 on error.

Category:

Tagging Functions
