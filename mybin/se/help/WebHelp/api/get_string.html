_str get_string(_str &line, _str prompt, _str arg_type_info='', _str initial_value='')

Prompts user for an input string on the command line. The result is placed in the variable line. The users response is initialized to initial_value if given. prompt is displayed to left of user input area. get_string supports retrieval based on the prompt argument and completion.

Parameters:

line - (output) set to value entered by user

prompt - the command prompt.

arg_type_info - completion info has the syntax arg_type[;help_name] and is used for argument completion and help. arg_type may be any completion constant listed in "slick.sh'". Completion constants have the suffix "_ARG". If arg_type starts with a '-' character, the arg_type is assumed to be the completion constant name without the "_ARG" suffix (i.e. "FILE" for FILE_ARG). If the arg_type starts with "-.", the string following is assumed to be a command name. The command name may be used to get the completion information and help. help_name may specify a help item in "vslick.hlp" (UNIX: "uvslick.hlp") or be a message. If help_name starts with a '.', it should specify a help item in "vslick.hlp" (UNIX: "uvslick.hlp").

initial_value - (optional) The command line is set to initial value. If you want to specify this parameter and you don't want to specify the third parameter, pass '' for the third parameter.

Returns: _str

Returns 0 if successful. Otherwise 1 (user aborted) is returned. On error, message is displayed.

Example:

     /* Completion/help on edit command. */
     status=get_string(line,'Filename: ', '-.edit')

     /* Completion/help on edit command. */
     status=get_string(line,'Filename: ', FILE_ARG';.edit command')

     /* Completion on file argument with pop-up message help. */
     status=get_string(line,'Filename: ','-FILE;Help message')

     /* Completion on file argument with pop-up message help. */
     status=get_string(line,'Filename: ','-.edit;Help message')

Command Line Functions, Keyboard Functions
