_command void ispf_hilite(_str arglist="")

This command is used to specify the use of color coding in the editor. Normally, color coding is automatically configured when the file is opened. The standard ISPF hilite options for SEARCH, PAREN, and LOGIC coloring are not supported.

Syntax:

   HILITE [ ON | OFF | DEFAULT | OTHER | ASM | BOOK | C | COBOL | DB2 | DTL |
            HTML | JAVA | JCL | PANEL | PASCAL | PLI | REXX | SKEL | IDL]
          [RESET] [FIND] [CURSOR] [DISABLED]
Arguments:
ON
Turns on program color coding.
OFF or DISABLE
Turns off program color coding.
DEFAULT or OTHER
Select fundamental mode color coding
RESET
Reset options, turns color coding on.
ASM
System/390 Assembler Language
BOOK
BookMaster (not supported)
C
C or C++
COBOL
COBOL
DB2
DB2 sql
DTL
Dialog Tag Language (not supported)
HTML
Hypertext Markup Language
JAVA
Java
JCL
MVS Job Control Language
PANEL
ISPF Panel Language (not supported)
PASCAL
Pascal
PLI
or pl1 PL/I
REXX
Rexx
SKEL
ISPF Skeleton Language (not supported)
IDL
Interface Definition Language
FIND
Toggles highlighting of text when you do a find, this does not work exactly like ISPF, it simply toggles the SlickEdit "Leave Selected" option (see configuration options, "Search Tab").
CURSOR
Toggles highlighting of the line containing the cursor. This also does not function exactly like ISPF, instead it simply toggles the "Draw Box around current line" option (see configuration options, "General Tab").

ISPF Primary Commands
