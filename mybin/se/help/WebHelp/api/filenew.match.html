_str filenew_match(_str name, boolean find_first)

Parameters:

name - file name to search for

find_first - 'true' to find first, 'false' to find next

Returns: _str

Returns next file name which is a prefix match of name. find_first must be non-zero to initialize matching. Returns '' when no more matches are found.

If there are absolute no matches to name, and find_first is true, it will return name.

This function behaves like f_match, but allows the user to type a non-existent file name. This is useful in order to get completion for directory names, but still be able to specify a new file.
