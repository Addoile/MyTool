int shell(_str program, _str options="", _str shell_processor="", int &process_id=null)

Runs the external program specified. The program parameter should include arguments to be passed to the program as well.

A positive or 0 return code indicates the return code is from the program executed. A negative return code indicates one of the OS or SlickEdit return codes listed in "rc.sh".

Parameters:

options - defaults to '' and may be one or more of the following:

'Q'
Run program without showing console window. DO NOT USE WITH 'W' OPTION.
'P'
Do not search for SlickEdit batch programs.
'A'
Run program asynchronously.
'W'
Run slkwait program to wait for a key press before closing application. DO NOT USE WITH 'Q' OPTION. Under UNIX, this argument is ignored if shell_processor is specified.
'N'
No command processor. Use this option when you notice an extra command shell window is created when you are trying to run a windows program. By default, if any command shell special characters (ex. '>', '<', '|'. There are others.) are found in the program argument, the command processor is used to process the program argument. This can cause a command shell window to be created when you are trying to run a windows program.
'B'
Allow user to break by pressing Ctrl+Alt+Shift (Windows only).

For UNIX, the shell_processor argument allows you to specify an alternate shell command line to parse the command string specified. For non-UNIX platforms, this argument is ignored. shell_processor defaults to "$SHELL -c". For backward compatibility with SlickEdit, if shell_processor does not contain a space, " -c" is concatenated on to the end of the string.

Example:

          shell("cl -c test.c >$errors.tmp");
          shell("man cc","","xterm -T "man" -e /bin/sh -c");
