_command int ispf_sort(_str arglist="")

The sort command sorts lines of data in a specified order.

Syntax:

   SORT [ line_range ] [X] [NX] sort_field_1 ... sort_field_n

Arguments:

line_range
Two numbers or labels, specifying the starting and ending range of lines to sort, inclusive.
X
Look for matches only in lines.
NX
Do not report matches in excluded lines.
sort_field_i
Field specifications for sorting data. See below.

Sort supports up to 100 sort fields. Each one can have the following options:

A
Sort data in ascending (non-decreasing) order.
D
Sort data in descending (non-decreasing) order.
I
Case insensitive sort. This option has no effect if the -F or -N options are specified.
E
Case sensitive sort (default). This option has no effect if the -F or -N options are specified.
-F
Sort data as file names (case sensitive on Unix).
-N
Sort data as integers.
col1
Starting physical column boundary for this field.
col2
Ending physical column boundary for the field.

Returns: int

Returns 0 if successful. Common return codes are 1 (you tried to sort the build window), NOT_ENOUGH_MEMORY_RC, TOO_MANY_SELECTIONS_RC, and INVALID_SELECTION_HANDLE_RC. On error, a message is displayed.

Category:

ISPF Primary Commands
