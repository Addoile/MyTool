int tag_list_symbols_in_context(_str prefix, _str search_class, int tree_wid, int tree_index, _str(&tag_files)[], _str search_file, int &num_matches, int max_matches, int pushtag_flags, int context_flags, boolean exact_match, boolean case_sensitive, typeless &visited=null, int depth=0, _str(&template_args):[]=null)

Match the given symbol based on the current context information. Order of searching is:

  1. local variables in current function
  2. members of current class, including inherited members
  3. globals found in the current file
  4. globals imported explicitely from current file
  5. globals, (not static variables), found in other files
  6. any symbol found in this file
  7. any symbol found in any tag file
Failing that, it repeats steps (1-6) with pushtag_flags set to -1, thus disabling any filtering, unless 'strict' is true. The current object must be an editor control or current buffer.

For synchronization, macros should perform a tag_lock_matches(true) prior to invoking this function.

Parameters:

prefix - symbol prefix to match

search_class - name of class to search for matches

tree_wid - window id of tree control to insert into, 0 indicates to insert into a match set

tree_index - tree index to insert items under, ignored if (treewid == 0)

tag_files - (reference to _str[]) tag files to search

search_file - file to search for matches in (for imports)

num_matches - (reference) number of matches

max_matches - maximum number of matches allowed

pushtag_flags - VS_TAGFILTER_*, tag filter flags

context_flags - VS_TAGCONTEXT_*, tag context filter flags

exact_match - exact match or prefix match (0)

case_sensitive - case sensitive (1) or case insensitive (0)

visited - hash table of prior results

depth - depth of recursive search

template_args - [optional] hash table of template arguments

Returns: int

0 on success, <0 on error.

Category:

Tagging Functions
