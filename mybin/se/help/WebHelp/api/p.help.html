_str p_help

Menu, Menu Item - Gets or sets help command. The help command is executed when F1 is pressed while the menu/menu item is selected. The help command is typically a help or popup_imessage command. When menus are displayed, there is no instance handle. Menu and menu item properties can only be accessed in menu resources. The _menu_get_state and _menu_set_state function may be used to get and set menu properties of displayed menus.

Other objects - Gets or sets a help string. Help is displayed when F1 is pressed or when a command button with a help string defined (p_help!='') that does not have an lbutton_up event handler is pressed. If the help string starts with a '?' character, the characters that follow are displayed in a message box. The help string may also specify a unique keyword in the "vslick.hlp" (UNIX: "uvslick.hlp") help file. The unique keywords for the help file are contained in the file "vslick.lst" (UNIX: "uvslick.lst"). In addition, you may specify a unique keyword for any windows help file by specifying a string in the format: keyword:help_filename.

When F1 is pressed the dialog manager performs the following search to find the help to display:

  1. If the active control has help defined, display the help.
  2. If the active form has help defined, display the help.
  3. Search for any control with help defined.

Applies To:

All Window Objects

Category:

All Windows Properties
