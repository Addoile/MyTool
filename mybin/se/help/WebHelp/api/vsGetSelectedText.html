int VSAPI vsGetSelectedText(int wid, char *pszBuf, int BufLen, int markid=-, int flags=-, const char *pEOLChars=a, int EOLCharsLen=-)

This function retrieves the selected text. pszBuf is set to "" (strlen(pszBuf)==0) iif there is no selection.

Parameters:

wid - Window id of editor control. 0 specifies the current object.

pszBuf - Buffer to receive selected text. Set to "" if there is no selected text.

BufLen - Number of characters allocated to pszBuf.

markid - mark_id is a selection handle allocated by the vsAllocSelection or vsDuplicateSelection function. A mark_id of -1 specifies the active selection or selection showing and is always allocated.

flags - Must be -1.

pEOLChars - Line separator characters. When this is set to 0, "\r\n" (UNIX: "\n") is always used. The buffer's line format setting is not used to determine the value to assign to pEOLChars. To specify Unix style line separator characters in a Windows environment, see the example below.

EOLCharsLen - Number of characters in pEOLChars. -1 means pEOLChars is null terminated.

Returns: int

Returns the number of characters in the selected text.

Example:

The only way to have vsGetSelectedText() use Unix style line separators in Windows is to specify pEOLChars and EOLCharsLen. 

vsGetSelectedText(0, szText, 256, nMarkId, -1, "\n", 1);

Applies To:

Edit Window, Editor Control

Categories:

Edit Window Methods, Editor Control Methods
