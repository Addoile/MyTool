_command typeless e(_str filenameArg='', typeless a2_flags='', _str auto_create_firstw_arg='')

Command line version of the gui_open command. Edits the file(s) specified. filespec may have wild card characters ( or ? for DOS). edit Command options can be set globally or per extension. The edit mode of each file loaded is set based on file extension.

The useful edit options are listed below. For a complete list of options, see load_files. A few options are only useful in SlickEdit macros.

+#command
Execute command on active buffer. For example, "s test.c +#100" places the cursor on line 100 of test.c.
+ or -L[C|Z]
Turn on/off load entire file switch. The optional C suffix count the number of lines in the file. The Z suffix counts the number of lines in the file and truncates the file if an EOF character is found in the middle of the file.
+ or -LF
+LF turns off loading entire file and turns on fast line count. This is the fastest way for SlickEdit to count the number of lines in a file. -LF turns off fast line counting.
+ or -LB
Turns on/off binary loading. All edit/save options which performed translations are ignored. For example, tab expansion or tab compression options are ignored. This allows "safe" editing of binary files even when edit/save file translation are on.
+ or -LE
Turns on/off show EOF option. When on, EOF characters at the end of DOS files are not stripped. This option is not supported when the +LZ option is given.
+ or -LN
Turns on/off show new line characters option. When on, new line characters are initially visible.
+ or -W
If the file or buffer is not already displayed in a window, insert a window for displaying the file or buffer.
+ or -S
Turn on/off unmodified block swapping to spill file.
+nnn
Load binary file(s) that follow in record width nnn.
+T [buf_name]
Start a default operating system format temp buffer with name buf_name.
+TU [buf_name]
Start a UNIX format temp buffer with name buf_name.
+TM [buf_name]
Start a MACINTOSH format temp buffer with name buf_name.
+TD [buf_name]
Start a DOS format temp buffer with name buf_name.
+Tnnn
Start a temp buffer where nnn is the decimal value of the character to be used as the line separator character.
+FU [buf_name]
Force SlickEdit to interpret a file in UNIX format.
+FM [buf_name]
Force SlickEdit to interpret a file in MAC format.
+FD [buf_name]
Force SlickEdit to interpret a file in DOS format.
+Fnnn
Force SlickEdit to use nnn as the decimal value of the line separator character.
+FTEXT
SBCS/DBCS mode. Open SBCS/DBCS file. This is the default mode when no mode (either SBCS/DBCS or Unicode) is specified.
+FEBCDIC
SBCS/DBCS mode. Open EBCDIC file.
+FUTF8
Unicode mode. Open UTF-8 file with or without signature.
+FUTF8S
Unicode mode. Open UTF-8 file with or without signature.
+FUTF16LE
Unicode mode. Open UTF-16 little endian file with or without signature.
+FUTF16LES
Unicode mode. Open UTF-16 little endian file with or without signature.
+FUTF16BE
Unicode mode. Open UTF-16 little endian file with or without signature.
+FUTF16BES
Unicode mode. Open UTF-16 big endian file with or without signature.
+FUTF32LE
Unicode mode. Open UTF-32 little endian file with or without signature.
+FUTF32LES
Unicode mode. Open UTF-32 little endian file with or without signature.
+FUTF32BE
Unicode mode. Open UTF-32 little endian file with or without signature.
+FUTF32BES
Unicode mode. Open UTF-32 big endian file with or without signature.
+FCPddd
Unicode mode. Open code page file specified by ddd. Under windows, this can be any valid code page or one of the VSCP_* constants defined in "slick.sh."
+FACP
Unicode mode. Open active code page file. Under windows, this can be any valid code page or one of the VSCP_* constants defined in "slick.sh."
+FAUTOXML
Unicode mode. Open XML file. The encoding is determined based on the encoding specified by the "?xml" tag. If the encoding is not specified by the "?xml", the file data is assumed to be UTF-8 data which is consistent with XML standards. We applied some modifications to the standard XML encoding determination to allow for some user error. If the file has a standard Unicode signature, the Unicode signature is assumed to be correct and the encoding defined by the "?xml" tag is ignored.
+FAUTOUNICODE
If the file has a standard Unicode signature, open the file as a Unicode file. Otherwise, the file is loaded as SBCS/DBCS data.
+FAUTOUNICODE2
If the file has a standard Unicode signature or "looks" like a Unicode file, open the file as a Unicode file. Otherwise, the file is loaded as SBCS/DBCS data. This option is NOT full proof and may give incorrect results.
+FAUTOEBCDIC
If the file "looks" like an EBCDIC file, open the file as an EBCDIC file. Otherwise, the file is loaded as SBCS/DBCS data. This option is NOT full proof and may give incorrect results. We have attempted to make this option support binary EBCDIC files.
+FAUTOEBCDIC,UNICODE
This option is a combination of "+FAUTOEBCDIC" and "+FAUTOUNICODE" options.
+FAUTOEBCDIC,UNICODE2
This option is a combination of "+FAUTOEBCDIC" and "+FAUTOUNICODE2" options.
+ or -E
Turn on/off expand tabs to spaces switch. Default is off. Tab increments of 8 are the default if not otherwise specified in the language setup for this file type.
+ or -D
Turn on/off memory buffer name search. Disk file search.
+B buf_name
Look in memory only for buffer with name buf_name. This MUST specify the exact contents of p_buf_name or p_DocumentName. DO NOT put quotes around buffer names with spaces. Use absolute() if necessary when searching for a fully qualified filename.
+N or -N
Network support switch. When on, SlickEdit will detect when another application has the same file open and automatically select "Read only" mode. This option requires an extra file handle open to the original file.
The options above that only show a plus sign will also take a minus sign. However their function will remain the same.

If you need to edit a file whose name contains space characters, place double quotes around the name.

Returns: typeless

Returns 0 if successful. Common return codes are FILE_NOT_FOUND_RC (occurs when wild card specification matches no files), NEW_FILE_RC (empty buffer created with filename specified because file did not exist), PATH_NOT_FOUND_RC, TOO_MANY_FILES_RC, and TOO_MANY_SELECTIONS_RC. On error, message is displayed.

Applies To:

Edit Window

Example:

e +l test
Loads all of the file test and frees the file handle. If test is already loaded, its buffer is activated.
e "this is.c"
Edit a file whose name contains a space character.
e +d test
Loads the file test from disk regardless if a copy already exists in memory.
e +w test
(One file per window option) Display file test in its own window. If file test is already displayed in a window, the window is activated.
e +b test
Activates the buffer test if it exists.
e +t test
Starts a new buffer called test.
e +40 +d test
Loads the binary file test from disk in 40 character line width. File may be edited and saved.
