_command typeless bind_to_key(_str commandName='')

Binds command specified to key(s) you type.

Prefix and cancel keys may only be bound by specifying the max_keystrokes parameter. max_keystrokes is a positive number representing the maximum number of keystrokes that will be pressed. The max_keystrokes parameter allows prefix keys to be rebound.

Use the create_prefix_key command to create a new prefix key for a multiple key sequence key binding. The -r and -m options may be used to select key bindings to the root or mode key table. If you don't quite understand what root and mode key tables are, you should not specify the -r or -m option.

Here are the reasons for selecting each option:
   -r You want this key binding to be active in all modes.
   -m You want this key binding to be active only in the current mode.

If the current mode is fundamental, the -r and -m options have no effect. Keys bound while in fundamental mode will be active in all modes unless there is a mode key binding.

In VI emulation, the -r option effects all modes and the -m option effects the current mode only which can be insert or command mode.

Returns: typeless

Returns 0 if successful. Possible return values are 1 (command not found) and COMMAND_CANCELLED_RC. On error, message is displayed.

Applies To:

Edit Window

Example:

cmdline is a string in the format: [-max_keystrokes] [-r | -m] command

Category:

Keyboard Functions

See Also:

unbind_key, list_keydefs, create_prefix_key
