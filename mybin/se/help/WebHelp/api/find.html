_command int find(_str string='', _str options=null, VSSEARCH_BOUNDS vssearch_bounds=null)

Syntax: /string [ /options ] or L /string [ /options ] or find /string [ /options ]

The /, L, or find command searches for the string specified. Press and hold Ctrl+Alt+Shift to terminate a long search. Any search or search and replace command may be terminated by pressing and holding Ctrl+Alt+Shift. If you use the L or find command, the first non blank character is used as the delimiter.

In ISPF emulation, this command is not called when invoked from the command line. Instead ispf_find is called. Use ("Edit", "Find...") to explicitly invoke the find command.

When no parameters are specified, you are prompted for the string parameter. While you enter the string parameter the following keys take on a new meaning:

Ctrl+R
Searches in reverse for the next occurrence of the search string
Ctrl+S
Searches forward for the next occurrence of the search string.
Ctrl+T
Toggles regular expression pattern matching on/off. See section Regular Expressions for information on regular expressions. The key bound to the BRIEF emulation command re_toggle will also toggle regular expression pattern matching.
Ctrl+W
Toggles word searching on/off. Word searching is useful for search and replacing variables.
Ctrl+C
Toggles case sensitivity. The key bound to the BRIEF emulation command case_toggle will also toggle the case sensitivity.
Ctrl+M
Toggles searching within selection.
Ctrl+O
Toggles incremental search mode.
Ctrl+Q
Quotes the next character typed.

Parameters:

string - Input string to search for.

options - The options argument is a string of one or more of the following:

E
Exact case.
I
Ignore case.
-
Reverse search.
M
Limit search to marked area.
<
If found, place cursor at beginning of word.
>
If found, place cursor at end of word.
R
Interpret string as a regular expression. See section SlickEdit Regular Expressions for syntax of regular expression.
U
Interpret string as a UNIX regular expression. See section UNIX Regular Expressions.
B
Interpret string as a Brief regular expression. See section Brief Regular Expressions.
L
Interpret string as a Perl regular expression.
&
Interpret string as a Wildcard regular expression.
H
Search through hidden lines.
#
Highlight matched occurrences with highlight color.
N
Do not interpret search string as a regular expression.
P
Wrap to beginning/end when string not found.
W
Limits search to words. Used to search for variables.
W
SlickEdit-regular-expression Specifies the valid characters in a word. The default value is [A-Za-z0-9_$].
W:P
search to word prefix. For example, searching for "pre" matches "pre" and "prefix" but not "supreme" or "supre".
W:PS
Limits search to strict word prefix. For example, searching for "pre" matches "prefix" but not "pre", "supreme" or "supre".
W:S
Limits search to word suffix. For example, searching for "fix" matches "fix" and "sufix" but not "fixit".
W:SS
Limits search to strict word suffix. For example, searching for "fix" matches "sufix" but not "fix" or "fixit".
Y
Binary search. This allows start positions in the middle of a DBCS or UTF-8 character. This option is useful when editing binary files (in SBCS/DBCS mode) which may contain characters which look like DBCS but are not. For example, if you search for the character 'a', it will not be found as the second character of a DBCS sequence unless this option is specified.
,
Delimiter to separate ambiguous options.
XCLetters'
Requires the first character of search string NOT be one of the color coding elements specified. For example, "XCS" requires that the first character not be in a comment or string. CCLetters is a string of one or more of the following color coding element letters:
O

Comment

L

1

2
Symbol 2
3
Symbol 3
4
Symbol 4
F
Function color
V
No save line
CCCLetters
Requires the first character of search string to be one of the color coding elements specified. See CCLetters above.
Default search options may be set by the Search Options ("Tools > Options > Editing > Search").

vssearch_bounds - Specifies ISPF search bounds parameters

Returns: int

Returns 0 if the search string specified is found. Common return codes are STRING_NOT_FOUND_RC, INVALID_OPTION_RC and INVALID_REGULAR_EXPRESSION_RC. On error, message is displayed

Applies To:

Edit Window, Editor Control

Example:

Command Line Examples

/xyz/-
Search for string "xyz" using default search case in reverse.
L /xyz/-
Search for string "xyz" using default search case in reverse. Notice that prefixing the string "/xyz/-" with L has no effect on this command.
L $xyz$-
Search for string "xyz" using default search case in reverse. The L command can change the string delimiter which in this case is not necessary.
L $/$
Search forward for the character slash. The L command has been used so that the string delimiter could be changed to $ which allows searching for the / character.
L $/$-
Search backward for the character slash.
/i/w
Search for the variable i.
/xyz/e
Search for "xyz" in exact case.
