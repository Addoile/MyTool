typeless _SetDialogInfoHt(_str key, typeless value, int wid=0, boolean usePUser2=false)

Store data to a dialog indexed by key. Use this instead of using multiple p_user variables for several controls.

Use _GetDialogInfoHt to retrieve data from a dialog by hash table key.

This function differs from _SetDialogInfo because it stores data in a hash table that is indexed by string key, rather than an array. Use _SetDialogInfo, _GetDialogInfo to store/retrieve data to/from a dialog by integer index.

IMPORTANT:
The old value stored by hash table key is returned. null is returned if there is no old value stored. You should initialize your key/value pairs (e.g. in an on_create event) if you want to safely take advantage of the return value.

Parameters:

key - A unique hash table key that represents a given item.

value -

wid - (optional). If specified, use this control's p_user instead of p_active_form. 0 specifies the active form. Defaults to 0.

usePUser2 - If true, store the data on the p_user2 property

Returns: typeless

Old value stored by key. null is returned if there is no old value stored for that key.

Category:

Form Functions
