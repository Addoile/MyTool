_str list_matches(_str name, typeless match_fun_index, _str title='', int multi_select=0, _str help_proc='', boolean auto_select_if_one_match=false, boolean doAddToAutoCompletion=false, typeless &words=null, boolean doAddToArgumentCompletion=false)

Creates a pop-up selection list of items that are a prefix match of name. Returns line selected or '' to indicate that nothing was selected. match_fun_name is a prefix name or index of a completion function. "_match" is appended to match_fun_name to determine the complete function name. The following are completion functions tag_match, b_match, c_match, _pic_match, _form_match, mt_match, e_match, f_match, _menu_match, m_match, pcb_match, pc_match, and v_match. New completion functions may be defined (see comment in module "complete.e" for more information). The selection list window title is set to title if given. The help_fun_name argument is the name of a help procedure.

Parameters:

name - Prefix string to match

match_fun_index - One ????_ARG constants defined in slick.sh OR index of find first/next function

title - Title of popup-window

multi_select - If Not '', allow multiple selections This argument is here for backward compatibility

help_proc - Name of help function to call

auto_select_if_one_match - Auto select if 1 match

Returns: _str

If allow_multi_select is not '', multiple lines in the selection list may be selected with the space- bar key. list_matches returns each selected line separated with a space or if there are too many, a list file specification is returned. A list file specification starts with the character '@'. If allow_multi_selection is not given or '', line selected is returned. '' is returned to indicate user pressed cancel key, list is null, or error. On error message is displayed and user is prompted to press a key.

Example:

       file_name=list_matches('','f');
       if (file_name=='' ) {
            return(1)  // Cancel key pressed or list null.
       }
       message('file_name='file_name);
