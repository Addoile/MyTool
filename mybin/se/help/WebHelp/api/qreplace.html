int qreplace(_str search_string, _str replace_string, typeless options, typeless search_mark="")

Searches from cursor position for search_string and replaces matches with replace_string. This procedure supports the same options as the replace command. Unlike the replace command, this procedure does not use any default options not specified by the options parameter and requires the search_string, replace_string, and options parameters to be given. The options argument is a string of one or more of the following:

E
Exact case.

I
Ignore case.

-
Reverse search.

M
Limit search to marked area.

V
Preserve case. When specified, each occurrence found is checked for all lower case, all upper case, first word capitalized or mixed case. The replace string is converted to the same case as the occurrence found excepted when the occurrence found is mixed case (possibly mulltiple capitalized words). In this case, the replace string is used without modification.

<
If found, position cursor at beginning of word.

>
If found, position cursor at end of word.

*
Make changes without prompting.

R
Interprets string1 to be a SlickEdit regular expression. String2 may specify tagged expressions with a pound followed by a group number 0-9. Count the left braces '{' to determine a group number. The first tagged expression is "#0". See SlickEdit Regular Expressions.

U
Interpret string as a UNIX regular expression. See section UNIX Regular Expressions. String2 may specify tagged expressions with a backslash followed by a group number 0-9. Count the left parenthesis '(' to determine a group number. The first tagged expression is "\1" and the last tagged expression is "\9". See UNIX Regular Expressions.

B
Interpret string as a Brief regular expression. See section Brief Regular Expressions. String2 may specify tagged expressions with a backslash followed by a group number 0-9. Count the left braces '{' to determine a group number. The first tagged expression is "\0". See Brief Regular Expressions.

H
Search through hidden lines.

N
Do not interpret search string as a regular search string.

P
Wrap to beginning/end when string not found.

W
Limits search to words. Used to search and replace variable names.

W=SlickEdit-regular-expression
Specifies the valid characters in a word. The default value is [A-Za-z0-9_$]

W:P
Limits search to word prefix. For example, searching for "pre" matches "pre" and "prefix" but not "supreme" or "supre".

W:PS
Limits search to strict word prefix. For example, searching for "pre" matches "prefix" but not "pre", "supreme" or "supre".

W:S
Limits search to word suffix. For example, searching for "fix" matches "fix" and "sufix" but not "fixit".

W:SS
Limits search to strict word suffix. For example, searching for "fix" matches "sufix" but not "fix" or "fixit".

Y
Binary search. This allows start positions in the middle of a DBCS or UTF-8 character. This option is useful when editing binary files (in SBCS/DBCS mode) which may contain characters which look like DBCS but are not. For example, if you search for the character 'a', it will not be found as the second character of a DBCS sequence unless this option is specified.

,
Delimiter to separate ambiguous options.

XCCLetters
Requires the first character of search string NOT be one of the color coding elements specified. For example, "XCS" requires that the first character not be in a comment or string. CCLetters is a string of one or more of the following color coding element letters:

O
Other
K
Keyword
N
Number
S
String
C
Comment
P
Preprocessing
L
Line number
1
Symbol 1
2
Symbol 2
3
Symbol 3
4
Symbol 4
F
Function color
V
No save line

CCCLetters
Requires the first character of search string to be one of the color coding elements specified. See CCLetters above.

If the '*' option is not specified, you will be prompted with the message "Yes/No/Last/Go/Quit?" for each occurrence of string1. Press one of the following keys to take an action:

'Y' or SPACE
Make change and continue searching.
'N' or BACKSPACE
No change and continue searching.
'L' or .
Make change and stop searching.
'G' or !
Make change and change the rest without prompting.
'Q' or ESC
Exits command. By default, the cursor is NOT restored to its original position. If you want the cursor restored to its original position, invoke the command "set-var def-restore-cursor 1" and save the configuration. Quit searching.
Ctrl+G
Exits command and restores cursor to its original position.
Ctrl+R
Searches in reverse for next occurrence of search string.
Ctrl+S
Searches forward for next occurrence of search string.
Ctrl+T
Toggles regular expression pattern matching on/off. The key bound to the BRIEF emulation command re_toggle will also toggle regular expression pattern matching. See section Regular Expressions for more information.
Ctrl+W
Toggles word searching on/off.
Ctrl+C
Toggles case sensitivity. The key bound to the BRIEF emulation command CASE_TOGGLE will also toggle the case sensitivity.
Ctrl+M
Toggles searching within mark.
F1 or '?'
Displays help on C command.

The optional orig_cursor_pos_mark argument is a selection handle which specifies the text position where the cursor position should be restored if Ctrl+G is pressed to terminate the search.

Returns: int

Returns 0 if search string found and search & replace not cancelled. Common return codes are COMMAND_CANCELLED_RC, STRING_NOT_FOUND_RC, TOO_MANY_SELECTIONS_RC, INVALID_REGULAR_EXPRESSION_RC, and INVALID_OPTION_RC. On error, message displayed. The global variable "_Nofchanges" is set to the number of occurrences changed.

Applies To:

Edit Window
