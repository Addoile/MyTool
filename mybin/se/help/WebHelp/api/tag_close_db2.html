int tag_close_db2(_str dbfilename=null)

Deprecated. Use tag_close_db() instead with caching options.

Close the given tag database (or the current one). This does not technically close the database, it only switches it to being open for read-only. This is useful to clean up after writing to a database. Use tag_close_db to permanently close a database.

Higher level than tag_close_db. Closes the database for write options, leaving it open for read options, caching anything that was buffered.

Parameters:

dbfilename - (optional) name of tag database, default is current

Returns: int

0 on success, <0 on error.

Since:

3.0
