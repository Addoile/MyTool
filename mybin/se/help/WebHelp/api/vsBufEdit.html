int VSAPI vsBufEdit(const char *pszFilename, const char *pszLoadOptions="", int IgnoreNotFound=a, const char *pszLoadOptions2="", int quiet=a)

Parameters:

pszFilename - File to open.

pszLoadOptions - One of the following values:

""
Display existing buffer or read pszFilename from disk.
"+d"
Always read pszFilename from disk even if this file is already loaded in the editor.
"+b"
Only succeed if a buffer already exists of the name pszFilename.

IgnoreNotFound - If non-zero, create a new buffer with name pszFilename if this file is not found. This option has no effect if pszLoadOptions is "+bi" or "+b".

pszLoadOptions2 - This option may be one of the following:

""
No effect.
"+L"
Always load entire file and close file handle.
"-L"
Never load entire file and leave file handle open to file.
"+E:ddd"
Expand tabs to spaces. This expands tabs in increments of ddd.
"+E"
Expand tabs to spaces. If your tabs settings for the file being loaded are of the form "+<increment>" (like +4), then tabs are expanded in increments of increment. Otherwise tabs are expanded in increments of 8. To set your tabs in a form "+<increment>", bring up the Extension Options dialog box ("Tools", "Options", "File Extension Setup...", select the Indent tab, select the extension, and set the "Tabs:" text box). For languages such as REXX and UNIX shell scripts which require the contents of the file to be analyzed before the file type is known, the fundamental mode tab settings are used.

Returns: int

Returns buffer id if successful (0 is a valid buffer id). Otherwise a negative error code is returned.
