void _xmlcfg_sort_on_attribute(int iHandle, int ParentNodeIndex, _str PrimaryAttrName, _str PrimaryOptions, _str PrimaryFolderElementName=null, _str SecondaryAttrName=null, _str SecondaryOptions=null)

Sorts nodes that are children of a specified node.

Parameters:

iHandle - XMLCFG tree handle returned from _xmlcfg_open, _xmlcfg_open_from_buffer, or _xmlcfg_create

ParentNodeIndex - Children of this node are sorted

PrimaryAttrName - Name of primary attribute to sort on.

PrimaryOptions - String of one or more of the following options:

F
Sort filenames
N
Sort numbers
I
Case insensitive sort. Sort is case sensitive if this option is not specified.
D
Descending. Sort is ascending if this option is not specified.
P
Place parent nodes at the top after sort. Here we consider a parent nodes with name PrimaryFolderElementName.
2
Specialized filename sort. Sort case insensitive on name without path, then case sensitive on name without path, then case insensitive on path.

PrimaryFolderElementName - Only used if P option specified in PrimaryOptions. Indicates the element of a folder.

SecondaryAttrName - Secondary attribute name to sort on

SecondaryOptions - Specifies secondary sort options. See primary options for more information.

Example:

// Sort the project files in the workspace 

int ProjectsNode=_WorkspaceGet_ProjectsNode(handle);
if (ProjectsNode>=0) {
   _xmlcfg_sort_on_attribute(handle,ProjectsNode,'File','2');
}

// Sort a folder node and place sub folders at the top.
// Note that it is OK that  nodes have no "N"
// attribute but do have a "Name" attribute.
_xmlcfg_sort_on_attribute(handle,FolderNode,"N",
   "Folder",
   "Name",
   "2P");
