_command int appmenu(typeless arg1="")

This command is used to execute a menu item of another application.

This command is only supported under Windows, Windows 95/98, and Windows NT.

cmdline is a string in the format: [-a] [-c[p] "ClassName"] [-t[p] "WindowTitle"] MenuSpec

Where MenuSpec is a list of menu items separated by a vertical bar character '|' which indicate how to traverse the menu to find the final menu item. The optional 'p' which follows the -c or -t options indicates that the name specified is a prefix match of the class or window title respectively. Use the -a option to make the other application active. Sometimes, the other application becomes active anyway (depending on the Version of the OS, whether you are creating a dialog box, and the other application).

Returns: int

0 if succesful

Example:

syntax: appmenu [-a] [-c[p] classname] [-t[p] windowtitle] <menuspec>

If you use the -t option, don't forget to use '"' around titles with spaces

menuspec:
Separate with "|".
    Will translate the & characters that may be in the menus, not "...".
    Ex.:"File|Open..."
appmenu -a -cp Afx -tp "Microsoft Developer Studio" File|Open...
appmenu -a -t "Borland C++" File|Open...
