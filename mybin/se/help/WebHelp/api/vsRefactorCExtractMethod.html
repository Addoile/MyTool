int VSAPI vsRefactorCExtractMethod(int handle, const char *filename, const char *symbolName, int &createMethodCall, seSeekPos startSeekPos, seSeekPos endSeekPos, char *returnType, int returnTypeMaxLength, char *paramInfo, int paramInfoMaxLength, int flags, int syntaxIndent)

Analyze what is necessary to do in order to extract the code between 'startSeekPos' and 'endSeekPos' into a separate function and replace the code with a call to the mew function.

Each line of parameter inforomation is a string of the form: 

   name [tab] return_type [tab] reference [tab] required [newline]

Parameters:

handle - Handle to transaction

filename - Filename to find symbol in

symbolName - Name of symbol to extract method from

createMethodCall - [output] < 0 if there is a problem creating a function call, 0 otherwise

startSeekPos - Seek position to begin extraction at

endSeekPos - Seek position to end extraction at

returnType - [output] set to return type of function

returnTypeMaxLength - number of bytes allocated to returnType

paramInfo - [output] set to list of function parameters

paramInfoMaxLength - number of bytes allocated to paramInfo

flags - Optional flags, bitset of VSREFACTOR_FORMAT_* (above)

syntaxIndent - Syntax indent amount

Returns: int

0 on success, <0 on error. If paramInfoLen is not long enough, this method will return INSUFFICIENT_MEMORY_RC, you should allocate at least 4K to paramInfoLen.

Category:

Refactoring Functions
