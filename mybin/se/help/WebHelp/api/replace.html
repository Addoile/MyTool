_command int replace(_str string1='', _str string2=null, _str options=null)

In ISPF emulation, the replace or c command is not called when invoked from the command line. Instead ispf_replace or ispf_c (short for ispf_change) command is invoked, respectively. Use ("Edit", "Replace...") to explicitly invoke the replace or c command.

The replace or c command changes occurrences of string1 with string2. If no parameters are specified, you are prompted for the string1 and string2 parameters. From a macro, the string1, string2, and options parameters may be passed as separate arguments which do not require any delimiters. While you enter the string1 parameter the following keys take on a new meaning:

Ctrl+R
Searches in reverse for the next occurrence of the search string.

Ctrl+S
Searches forward for the next occurrence of the search string.

Ctrl+T
Toggles regular expression pattern matching on/off. See section Regular Expressions for information on regular expressions. The key bound to the BRIEF emulation command re_toggle will also toggle regular expression pattern matching.

Ctrl+W
Toggles word searching on/off.

Ctrl+C
Toggles case sensitivity. The key bound to the BRIEF emulation command case_toggle will also toggle the case sensitivity.

Ctrl+M
Toggles searching within mark.
Ctrl+O
Toggles incremental search mode.
Ctrl+Q
Quotes the next character typed.

The First non blank character is used as the delimiter. The options argument is a string of one or more of the following:

E
Exact case.
I
Ignore case.
-
Reverse search.
M
Limit search to marked area.

V
Preserve case. When specified, each occurrence found is checked for all lower case, all upper case, first word capitalized or mixed case. The replace string is converted to the same case as the occurrence found excepted when the occurrence found is mixed case (possibly mulltiple capitalized words). In this case, the replace string is used without modification.

$
Replaced occurrences are highlighted with modified color.

<
If found, position cursor at beginning of word.
>
If found, position cursor at end of word.
*
Make changes without prompting.

R
Interprets string1 to be a SlickEdit regular expression. In addition, the characters \ and # take on new meaning in the replace string. See SlickEdit Regular Expressions.

U
Interpret string as a UNIX regular expression. See section UNIX Regular Expressions.
B
Interpret string as a Brief regular expression. See section Brief Regular Expressions.
H
Search through hidden lines.
N
Do not interpret search string as a regular search string.
P
Wrap to beginning/end when string not found.
W
Limits search to words. Used to search and replace variable names.

W=SlickEdit-regular-expression
Specifies the valid characters in a word. The default value is [A-Za-z0-9_$]

W:P
Limits search to word prefix. For example, searching for "pre" matches "pre" and "prefix" but not "supreme" or "supre".

W:PS
Limits search to strict word prefix. For example, searching for "pre" matches "prefix" but not "pre", "supreme" or "supre".

W:S
Limits search to word suffix. For example, searching for "fix" matches "fix" and "sufix" but not "fixit".

W:SS
Limits search to strict word suffix. For example, searching for "fix" matches "sufix" but not "fix" or "fixit".

Y
Binary search. This allows start positions in the middle of a DBCS or UTF-8 character. This option is useful when editing binary files (in SBCS/DBCS mode) which may contain characters which look like DBCS but are not. For example, if you search for the character 'a', it will not be found as the second character of a DBCS sequence unless this option is specified.

,
Delimiter to separate ambiguous options.

XCCLetters
Requires the first character of search string NOT be one of the color coding elements specified. For example, "XCS" requires that the first character not be in a comment or string. CCLetters is a string of one or more of the following color coding element letters:

O
Other
K
Keyword
N
Number
S
String
C
Comment
P
Preprocessing
L
Line number
1
Symbol 1
2
Symbol 2
3
Symbol 3
4
Symbol 4
F
Function color
V
No save line

CCCLetters
Requires the first character of search string to be one of the color coding elements specified. See CCLetters above.

If the '*' option is not specified, you will be prompted with the message "Yes/No/Last/Go/Quit?" for each occurrence of string1. Press one of the following keys to take an action:

'Y' or SPACE
Make change and continue searching.

'N' or BACKSPACE
No change and continue searching.

'L' or .
Make change and stop searching.

'G' or !
Make change and change the rest without prompting.

'Q' or ESC
Exits command. By default, the cursor is NOT restored to its original position. If you want the cursor restored to its original position, invoke the command "set-var def-restore-cursor 1" and save the configuration.

Ctrl+G
Exits command and restores cursor to its original position.

Ctrl+R
Searches in reverse for next occurrence of search string.

Ctrl+S
Searches forward for next occurrence of search string.

Ctrl+T
Toggles regular expression pattern matching on/off. The key bound to the BRIEF emulation command re_toggle will also toggle regular expression pattern matching. See section Regular Expressions for more information.

Ctrl+W
Toggles word searching on/off.

Ctrl+C
Toggles case sensitivity. The key bound to the BRIEF emulation command case_toggle will also toggle the case sensitivity.

Ctrl+M
Toggles searching within mark.

F1 or '?'
Displays help on replace command.

No space character is needed between the 'C' and the '/' delimiter UNLESS you use a delimiter other than '/'. You may use any character except space as the delimiter. For example, "c $/$\$" replaces occurrences of forward slashes to back slashes.

Command line examples:

c/x/y/m
Replace occurrences of x in the marked area with y using default search case sensitivity.

c $x$y$m
Replace occurrences of x in the marked area with y using default search case sensitivity. The string delimiter $ has been used requiring a space character after the C.

c/x/y/e*
Replace lower case occurrences of x with y without prompting.

c/i/something_more_meaningful/w
Replace occurrences of the variable i with something_more_meaningful

c/i/j/w=[A-Za-z]
Replace occurrences of the word i with j and specify valid characters in a word to be alphabetic.

c/{if|while}/x\0y\1/r
Replaces occurrences of if and while with xify and xwhiley. Unmatched groups are null. Note the \1 is replaced with null.

c $/$\\$r
Replace forward slash with backslash. Two backslashes represent single backslash for regular expression search and replace.

Returns: int

Returns 0 if search string and replace not cancelled. Common return codes are COMMAND_CANCELLED_RC, STRING_NOT_FOUND_RC, TOO_MANY_SELECTIONS_RC, INVALID_REGULAR_EXPRESSION_RC, and INVALID_OPTION_RC. On error, message is displayed. The universal scope variable "_Nofchanges" is set to the number of occurrences changed.

Example:

c/string1/string2[/options] or replace /string1/string2[/options]

Applies To:

Edit Window, Editor Control
