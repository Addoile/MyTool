int _select_type(_str markid="", _str option='T', _str newValue=null)

Parameters:

mark_id - is a selection handle allocated by the _alloc_selection built-in. A mark_id of '' specifies the active selection or selection showing and is always allocated. The third parameter is only supported by the 'T' option.

option - may be 'T', 'S', 'P', 'I', or 'U'.

If a value of 'T' is specified for option, one of the selection types "BLOCK", "CHAR", "LINE" or "" is returned. If the third parameter is specified, the selection style is set to new_type which must be "BLOCK", "CHAR", or "LINE". A null string is returned if the selection specified has not been set.

When a value of 'S' is specified for option, the select style is returned. Select styles are 'C' or 'E' which correspond to cut/paste (selection extends as the cursor moves) or begin/end respectively. A null string is returned if the selection specified has not been set.

If a value of 'P' is specified for option, selection anchor (pivot point) information is returned. The selection anchor information is a two letter string and is used to determine how the selected area moves as the cursor moves. This option is only useful for cut/paste style selections which extend the selection as the cursor moves. The first letter is 'B' if the begin mark is the line anchor or 'E' if it is not. The second letter is 'B' if the begin mark is the column anchor or 'E' if it is not. The null string is returned if the selection specified has not been set.

If a value of 'I' is specified for option, a '1' is returned if the selection specified is an inclusive selection. A '0' is returned if the selection specified is non-inclusive. A null string is returned if the selection specified has not been set. Currently this style affects only character selections.

When a value of 'U' is specified for option, a 'P' is returned if the selection specified is a persistent mark. Otherwise, '' is returned.

Returns: int

Returns the type or style information about the selection specified.
