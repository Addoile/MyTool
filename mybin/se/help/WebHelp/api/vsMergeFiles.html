int VSAPI vsMergeFiles(const char *pszBaseFilename="", int iBaseBufferId=-, int iBaseWindowId=a, const char *pszRev1Filename="", int iRev1BufferId=-, int iRev1WindowId=a, const char *pszRev2Filename="", int iRev2BufferId=-, int iRev2WindowId=a, const char *pszOutputFilename="", int iOutputBufferId=-, int iOutputWindowId=a, int iFlags=a, const char *pszCopy1Caption="", const char *pszCopy2Caption="", const char *pszCopy1AllCaption="", const char *pszCopy2AllCaption="", const char *pszImaginaryLineCaption=a)

Performs a three way merge and displays the results interleave or in a merge dialog.

Parameters:

pszBaseFilename - Name of the base file. This will be loaded from disk. If there is a buffer by this name in memory, it will be used.

iBaseBufferId - Buffer index for the base buffer. If you use this, be sure to specify "" for pszBaseFilename, and 0 for iBaseWindowId. Specify -1 to have this parameter ignored.

iBaseWindowId - Window Index for the first buffer. If you use this, be sure to specify "" for pszBaseFilename, and -1 for iBuf1Index. Specify 0 to have this parameter ignored.

pszRev1Filename - Name of the revision 1 file. This will be loaded from disk. If there is a buffer by this name in memory, it will be used.

iRev1BufferId - Buffer index for the revision 1 buffer. If you use this, be sure to specify "" for pszRev1Filename, and 0 for iRev1WindowId. Specify -1 to have this parameter ignored.

iRev1WindowId - Window Index for the revision 1 buffer. If you use this, be sure to specify "" for pszRev1Filename, and -1 for iBuf1Index. Specify 0 to have this parameter ignored.

pszRev2Filename - Name of the revision 2 file. This will be loaded from disk. If there is a buffer by this name in memory, it will be used.

iRev2BufferId - Buffer index for the revision 2 buffer. If you use this, be sure to specify "" for pszRev2Filename, and 0 for iRev2WindowId. Specify -1 to have this parameter ignored.

iRev2WindowId - Window Index for the revision 2 buffer. If you use this, be sure to specify "" for pszRev2Filename, and -1 for iBuf2Index. Specify 0 to have this parameter ignored.

pszOutputFilename - Name of the output 2 file.

iOutputBufferId - Buffer index for the output buffer. If you use this, be sure to specify "" for pszOutputFilename, and 0 for iOutputWindowId. If you specify iOutputBufferId or iOutputWindowId, the output will be left open after merge terminates(if there are conflicts) so that you can deal with the output. Specify -1 to have this parameter ignored.

iOutputWindowId - Window Id for the output buffer. If you use this, be sure to specify "" for pszOutputFilename, and -1 for iOutputBufferId. If you specify iOutputBufferId or iOutputWindowId, the output will be left open after merge terminates (if there are conflicts) so that you can deal with the output. Specify 0 to have this parameter ignored.

iFlags - Combination of the following flags:

VSMERGE_OPT_BASE_BUFFER
pszBaseFilename is a buffer name and not a relative filename which needs the path to be fully resolved.
VSMERGE_OPT_REV1_BUFFER
pszRev1Filename is a buffer name and not a relative filename which needs the path to be fully resolved.
VSMERGE_OPT_REV2_BUFFER
pszRev2Filename is a buffer name and not a relative filename which needs the path to be fully resolved.
VSMERGE_OPT_OUTPUT_BUFFER
pszRev2Filename is a buffer name and not a relative filename which needs the path to be fully resolved. Buffer is left open after merge for user to deal with.
VSMERGE_OPT_SMART
Feeds some conflict pieces back to diff in order to try to resolve conflicts. In many cases, will not make a difference IF YOU WANT YOUR USERS TO MAKE MOST OF THE DECISIONS WITH CONFLICTS, THIS OPTION IS NOT FOR YOU.
VSMERGE_OPT_INTERLEAVED
Gives the output in a single buffer with collisions labeled rather than in an interactive dialog.
VSMERGE_OPT_QUIET
Quiet option. Shuts off "Merge complete x conflicts" message.
VSMERGE_OPT_CALLERSAVES
When the user is prompted to save the file on close, does not actually save.
VSMERGE_OPT_FORCECONFLICT
Merge will never automatically copy a change/delete into the output. Instead, a conflict will be generated.
VSMERGE_OPT_SHOWCHANGES
In cases where merge automatically copies lines into the output, the lines are still colored with the color of the file that they came from so the user can see what happened.
VSMERGE_OPT_INDIVIDUALCONFLICTUNDO
Rather than use SlickEdit's buffer undo, when the undo button is pressed, the current conflict region is returned to its original state.
VSMERGE_OPT_IGNORESPACES
Does not force conflicts because of lines that differ in spacing. In cases where this happens, lines copied to output will always come from the base file for consistency.

pszCopy1Caption - This is the caption of the "Copy 1>>" button.

pszCopy2Caption - This is the caption of the "Copy 2>>" button.

pszCopy1AllCaption - This is the caption of the "Copy 1 All>>" button.

pszCopy2AllCaption - This is the caption of the "Copy 2 All>>" button/.

pszImaginaryLineCaption - This is the text used for imaginary lines. Note that this parameter is a global setting and will effect other diffs currently in progress. For this reason, you must make sure that this parameter is the same for all calls to vsDiffFiles and vsMergeFiles.

Returns: int

Returns 0 if successful and there are no conflicts. 1 if there are conflicts and the user chooses to save the file. 2 if there are conflicts and the user chooses not to save the file. A negative return code is returned and a error message box is diplayed if a file I/O error occurs or a file does not exist.

Category:
