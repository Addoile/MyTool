int VSAPI vsCreateEditorCtl(int wid, VSSYSHWND hwndparent, int x, int y, int width, int height, int BorderStyle=VSBDS_FIXED_SINGLE, int visible=a, int buf_id=-, void *pdata=a, int mdi_wid=a, int reserved1=a, void *pvwidget=a)

This function creates a new editor control. DO NOT use this function to create an editor control on a Slick-C® dialog box. If you are NOT calling this function during the VSINIT.pfnMDICreateEditWindow callback, you need to call vsRefresh with the 'A' option so that the scroll bars are updated.

You do not have to call vsDestroyEditorCtl on Windows because SlickEdit gets the Windows WM_DESTROY message and can cleanup their. However, under X windows you must call vsDestroyEditorCtl just before destroying the parent window which contains the editor control window. Otherwise, Visual SlickEdit will crash.

Parameters:

wid - Window id of new editor control. This should be 0 except when this function is called during the VSINIT.pfnMDICreateEditWindow callback.

hwndparent - System window handle of parent.

x, - y, width, height Size in pixels to use when created new editor control.

BorderStyle - Border style of editor control.

visible - When non-zero, editor control is created initially visible. Set the VSP_VISIBILE property to make an editor visible or invisible.

buf_id - Buffer id to initially attatch to the new editor control.

pdata - Initial window specific user data. Use vsWinQData function to get your window specific user data. Use the vsWinSetData function to change this value later.

mdi_wid - Window id of MDI frame window. If you have implemented our MDI callbacks, specify this parameter to register a new editor control MDI child. Use this parameter during the VSINIT. pfnMDICreateEditWindow callback. Currently only one MDI frame window (VSWID_MDI) is supported.

Returns: int

If successful, returns window id of new editor control. Otherwise negative error code is returned. 0 is not a valid window id and is never returned by this function.
