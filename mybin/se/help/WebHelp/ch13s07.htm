Advanced Help Configuration

This section outlines the ways that alternative and multiple Help file access can be configured.

F1 Index Help

F1 Help, also known as Word Help or SDK Help, searches external Help files for information about a symbol. For C++ compiler packages, this includes help on the Windows API, Standard C library, and Microsoft C Foundation Classes (if you have Microsoft Visual C++).

To use F1 Help, first configure the Help files to be used. You can also configure access to multiple Help files (such as MSDN Help). See Configuring Help Files.

After configuration, to search the Help files for the keyword at the cursor, use one of the following methods:

  • Press F1

  • Or, from the main menu, click Help → F1 Index Help

  • Or, on the SlickEdit command line, use the help_index command

  • Or, in any emulation, click the Question Mark button () located on the SlickEdit toolbar.

This will invoke the Help Index dialog box which scans a help .idx file for Help files that contain a prefix match or exact match of the Help keyword specified. You can specify other keywords by directly typing them into the Help Keyword field on this dialog.

UNIX, Linux, and Mac OS X F1 Help

UNIX, Linux, and Mac OS X users can get man page Help on the word at the cursor by pressing F1 (executes the wh command). When you are editing a Slick-C® macro, pressing F1 brings up macro language Help for the word at the cursor. You will find this especially useful for getting help on macro source generated during macro recordings. F1 works for the default CUA emulation and most other emulations as well.

Configuring Help Files

The Help system uses Help files to determine the Help information that is displayed. To add or remove Help files from SlickEdit®, from the main menu click Tools → Options → Help Options → F1 Index File. These options let you add an IDX file, which is typically used for providing Word Help on multiple Help files that may have duplicate Help index entries. If you want your IDX file to be used when you press Ctrl+F1 (wh command), make sure the Word help filename(s) box on the General Help options screen (Tools → Options → Help Options → General) specifies this file name.

Building the Help Index File vslick.idx

  1. From the main menu, click Tools → Options → Help Options → F1 Index File.

  2. Click Scan to automatically add Help files (.hlp) on the PATH. This should also find Microsoft Visual C++, C++ Builder, and Delphi Help files.

  3. If scanning did not pick up some Help files you want, add the other Help directories manually to the Help file path text box. Click Scan again.

  4. Click OKto build the vslick.idx file.

Usually, you only need one Help index file for all the compiler packages. However, you may prefer to separate the Help files into groups that you can access on different keys.

Changing the Default Word Help File(s)

To change the default Word Help file name(s) for Ctrl+F1 and Ctrl+F2 (wh and wh2 commands), complete the following steps:

  1. To set the Word Help file name, from the main menu, click Tools → Options → Help Options → General.

  2. In the Word help filename(s) box, specify the Help files for the wh command (Ctrl+F1), each separated with a plus sign. You can only specify one IDX or MVB file per command.

  3. Use a semicolon to indicate the start of the Help files for the wh2 command (Ctrl+F2) and specify its Help files separated with a plus sign.

  4. You can use another semicolon to indicate the start of the Help files for the wh3 command; however, this command is not bound to a key.

Help files have the extension .mvb, .hlp, or .idx. Do not use a plus sign to specify more than one IDX or MVB file. For example:

  • CtrlF1a.hlp + CtrlF1b.hlp; CtrlF2.hlp

  • vslick.idx; win31wh.hlp+mscxx.hlp+errors.hlp+vslick.hlp

  • CtrlF1.idx; CtrlF2.idx

  • vslick.idx; c:\msdev\help\vcbks40.mvb

Environment variables may be embedded in the Word Help file name(s) string by the following notation:

        %(
        envvar-name
        )

Configuring F1 MSDN Help

You can configure F1 Help to display information from Microsoft's MSDN Help system when searching for help in a Microsoft solution or project. Any MSDN library that is installed with Visual Studio 2003 and later installs this Help viewer. To set this configuration, from the main menu, click Tools → Options → Help Options → F1 MSDN Help, then select Use MSDN help for F1 help to enable the feature. Installed Collections is a list of the Help collections that are installed on your computer. By default, Help searches the collections with the word "combined" in the name, first. It searches in the index list as well as the keywords.
